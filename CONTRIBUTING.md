# contributing

## development

build: `make build`
test: `make test`
pre-commit: `make precommit`

## workflow

1. write tests before implementation
2. get tests reviewed before implementing
3. run `make precommit` before committing
4. update ROADMAP.md when starting/completing tasks
5. update DESIGN.md after architectural changes
6. update README.md after user-visible changes
